# 学习文档输出规则

> 本规则定义每次学习会话后的文档输出结构和格式。

---

## 一、文档清单

| 文档 | 用途 | 何时更新 |
|---|---|---|
| `ruler.md` | 本文件，输出规则 | 规则变更时 |
| `QA.md` | 小测验问答 + 评价 + 简要概括 | 每次测验回答后 |
| `top-知识扩展.md` | 详细知识扩展（尤其是用户主动询问的） | 每次有知识补充时 |
| `REVIEW_学习评分档案.md` | 学习评分跟踪 | 每讲结束时 |
| `0X_阶段X.X_讲解与评价.md` | 讲解原文存档 | 每讲生成时 |

---

## 二、QA.md 输出规则

### 2.1 结构

每道题按以下格式写入：

```markdown
### Q{编号}：{题目}

**来源阶段**：{阶段编号}

**用户回答**：
> {用户回答原文或摘要}

**评价**：
| 论点 | 评价 | 说明 |
|---|---|---|
| ... | ... | ... |

**总分**：{分数}

**补充要点**：
- {遗漏或需要记住的关键点，1-2 句}

**详细扩展**：
> 见 [top-知识扩展.md#{锚点}](top-知识扩展.md#{锚点})
```

### 2.2 要求

- 评价表格必须引用代码证据（文件路径 + 行号）
- 每题必须有"补充要点"，指出用户遗漏的最重要的一点
- 如果该题涉及知识扩展，在"详细扩展"字段指向 `top-知识扩展.md` 的具体锚点
- 回答简短，重点突出；详细内容全部在 `top-知识扩展.md` 中

---

## 三、top-知识扩展.md 输出规则

### 3.1 结构

每个知识扩展按以下格式写入：

```markdown
## {主题名称}

> 来源：{阶段编号} / {触发原因：用户主动询问 / 教学补充}

### 一句话

{一句话概括}

### 详细内容

{完整的知识讲解，包括代码示例、对比表格、架构图等}

### 与本项目的关联

{说明这个知识点在本项目中的应用位置和原因}

---
```

### 3.2 要求

- 每个知识扩展必须有唯一的一级标题，方便 QA.md 通过锚点跳转
- 代码示例必须可运行或有明确标注（伪代码/示意）
- 对比类内容用表格
- 架构类内容用 ASCII 图或文本图
- 每个扩展末尾必须有"与本项目的关联"段落
- 用户主动询问的内容优先级高于教学补充，放在文件前部

---

## 四、每阶段学习流程

每个阶段按以下固定节奏推进：

```
1. 情景预热（Scenario Warm-up）
   → 设定 1-3 个与下一阶段相关的情景题
   → 不查代码，凭已有认知先答
   → 目的：暴露认知盲区、激发好奇心、定位学习侧重

2. 讲解（Lecture）
   → 用类比/直觉讲核心概念
   → 给 2-3 处代码证据锚点
   → 由浅入深，先全局再局部

3. 提问巩固（Quiz）
   → 抛出 2-4 个启发问题
   → 用户不查代码先答
   → 覆盖"是什么 / 为什么 / 如果换一种方案会怎样"

4. 评价 + 纠偏（Evaluate）
   → 逐条评价用户回答，引用代码证据
   → 指出正确 / 错位 / 盲点
   → 必要时补充正确版本

5. 知识扩展（Expansion）
   → 用户主动询问的优先、详细
   → 教学补充的按需
   → 写入 top-知识扩展.md

6. 落地文档（Document）
   → QA.md：测验问答 + 简要评价 + 锚点链接
   → top-知识扩展.md：详细知识扩展
   → REVIEW_学习评分档案.md：更新评分
```

### 情景预热题的要求

- 与该阶段的核心知识点直接相关
- 从"如果让你做，你会怎么设计"的角度提问
- 不要求正确答案，重在暴露思考方式
- 题目数量：1-3 个，不超过阶段总时长的 20%

### 提问时的上下文提供

提问情景题时，如果涉及项目中的图结构、字段设计、模块划分等内容，**必须先给出相关信息**（图、字段列表、文件结构等），让用户能针对具体设计回答，而非凭空猜测。这也同时帮助用户巩固框架和字段设计。

---

## 五、整体要求

1. **QA.md 简要，top-知识扩展.md 详细**：QA.md 只放概括和评价，详细讲解在知识扩展中
2. **锚点可跳转**：QA.md 中引用知识扩展时，必须用 Markdown 锚点链接
3. **按时间追加**：新内容追加到文件末尾，不修改历史内容
4. **代码证据优先**：评价和知识点必须引用项目中的具体文件和行号
5. **每次学习会话后立即更新**：测验回答后立即写入 QA.md 和知识扩展
