  Learning Agent 项目分析

  一、项目定位

  这是一个基于 费曼学习法 的 AI 学习辅助系统，核心理念是通过「诊断 → 讲解 → 复述检测 → 追问 →
  总结」五步闭环帮助用户深入理解知识。当前版本 v0.2。

  三、架构分层

  CLI (repl.py) / HTTP API (FastAPI)
           ↓
     AgentService（会话编排层）
      ├─ agent_runtime（意图路由 / 计划 / 评估）
      ├─ LLM Service（模型调用）
      └─ LangGraph（状态图执行引擎）
           ↓
     存储层
      ├─ session_store（会话状态，memory/sqlite）
      ├─ learning_profile_store（学习档案，memory/sqlite）
      └─ personal_rag_store（个人 RAG 记忆，JSONL）

  四、核心流程

  1. 费曼学习闭环（graph.py）

  分三段子图执行：
  - 阶段 A：diagnose → explain（诊断先验知识 + 费曼式讲解）
  - 阶段 B：restate_check → followup（复述检测 + 追问漏洞）
  - 阶段 C：summarize（总结 + 生成复习建议）

  每次 /chat 请求只执行当前阶段对应的子图，靠 stage 字段在多轮间推进。

  2. 意图路由（agent_runtime.py）

  每次用户输入先经过 LLM 路由（失败则 fallback 到关键词规则），分四个分支：
  - teach_loop → 进入费曼教学主线
  - qa_direct → 直接回答问题（不推进阶段）
  - replan → 重新生成学习计划
  - review → 复盘

  3. 主题管理

  AgentService._detect_topic() 通过 LLM 自动识别主题切换，支持：
  - 主题切换时自动快照旧主题上下文（topic_segments）
  - 新主题从 stage=start 重新开始
  - 跨主题对比模式（comparison_mode）

  4. 学习档案沉淀（learning_analysis.py）

  总结阶段结束后自动执行：
  - 会话总结存储
  - 掌握度评分（基于关键词规则打分 0-100）
  - 错因归纳（关键词匹配提取标签）
  - 复习计划生成（根据分数决定间隔复习时间）
  - 长期记忆写入（topic_memory_entries + personal_rag JSONL）

  六、值得关注的设计特点

  1. 双入口：HTTP API + CLI REPL 共用同一个 AgentService，代码复用好
  2. 存储后端可切换：SESSION_STORE_BACKEND=memory|sqlite，通过配置切换
  3. 分支决策可观测：branch_trace 记录每次路由/评估/规划决策，/trace 命令可查
  4. Topic Segment 机制：主题切换时保留旧主题快照，支持跨主题上下文延续
  5. 个人 RAG：基于 JSONL 的简易关键词匹配检索，为后续 embedding 升级预留

  八、项目成熟度评估

  - 已完成：费曼闭环骨架、多轮会话、意图路由、学习档案存储与查询、CLI + API 双入口、worklog 留痕体系
  - 待完善：LLM 评估替代规则评估、真正的动态规划、embedding RAG、PostgreSQL 迁移、可观测体系
  (LangSmith/OTel)、生产级错误处理

  总体来说，这是一个结构清晰、分层合理的 v0.2
  原型，核心教学闭环和数据沉淀链路已跑通，适合作为下一步迭代的基础。最需要优先解决的是将关键词规则评估升级为 LLM      
  评估，以及修复 profile API 的路由冲突。
