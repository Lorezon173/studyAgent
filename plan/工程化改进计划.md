## 3. 详细改进方案

### 3.1 CI/CD 流水线 (P0)

**目标:** 自动化代码检查、测试、构建

**技术选型:** GitHub Actions

**流水线设计:**

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/ruff-action@v1
      - run: ruff check app/ tests/

  type-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install mypy
      - run: mypy app/ --ignore-missing-imports

  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
      - run: pip install uv
      - run: uv sync
      - run: uv run pytest tests/ -v --cov=app --cov-report=xml
      - uses: codecov/codecov-action@v4
        with:
          files: coverage.xml

  build:
    needs: [lint, type-check, test]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build Docker image
        run: docker build -t learning-agent:${{ github.sha }} .
```

**预期收益:**
- 每次提交自动检查代码质量
- PR 合并前强制通过所有检查
- 测试覆盖率可视化

### 3.6 密钥管理优化 (P2)

**目标:** 避免密钥明文存储

**方案对比:**

| 方案 | 复杂度 | 安全性 | 适用场景 |
|------|--------|--------|----------|
| 环境变量 | 低 | 中 | 开发环境 |
| .env 文件 (gitignore) | 低 | 中 | 开发环境 |
| HashiCorp Vault | 高 | 高 | 生产环境 |
| AWS Secrets Manager | 中 | 高 | AWS 部署 |
| Docker Secrets | 中 | 高 | Docker Swarm |

**推荐方案:**

```python
# app/core/secrets.py
import os
from functools import lru_cache

@lru_cache
def get_secret(key: str) -> str:
    """从环境变量或文件获取密钥"""
    # 优先从环境变量获取
    value = os.getenv(key)
    if value:
        return value
    
    # 从 Docker Secrets 文件获取
    secret_file = f"/run/secrets/{key.lower()}"
    if os.path.exists(secret_file):
        return open(secret_file).read().strip()
    
    raise ValueError(f"Secret {key} not found")
```

### 3.8 Makefile 标准化 (P2)

**目标:** 统一开发命令

```makefile
# Makefile
.PHONY: install dev test lint format build docker clean

install:
	uv sync

dev:
	uv run uvicorn app.main:app --reload --port 1900

test:
	uv run pytest tests/ -v --cov=app

lint:
	uv run ruff check app/ tests/
	uv run mypy app/

format:
	uv run ruff format app/ tests/

build:
	docker build -t learning-agent:latest .

docker:
	docker-compose up -d

clean:
	find . -type d -name "__pycache__" -exec rm -rf {} +
	find . -type f -name "*.pyc" -delete
	rm -rf .pytest_cache .coverage htmlcov

worker:
	celery -A app.worker.celery_app worker --loglevel=info
```
