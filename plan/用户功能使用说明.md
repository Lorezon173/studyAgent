# Learning Agent 用户功能使用说明（当前版本）

本文面向使用者，说明当前系统可用功能、典型操作流程，以及如何添加/检索 RAG 知识。

---

## 1. 功能总览

当前用户侧可用能力：

- 账号注册/登录（用户名+密码，自动分配数字 user_id）
- 学习对话（多阶段：讲解 -> 追问 -> 总结）
- 会话管理（按 session 继续学习）
- 学习档案查询（summary/mastery/errors/review）
- 技能查看（/skills）
- 知识库 RAG：
  - 入库：text/image（image 走 OCR）
  - 检索：global/personal 双轨
  - 个人轨道按 user_id 强隔离

---

## 2. 快速启动

### 2.1 启动后端

```bash
uv run uvicorn app.main:app --reload
```

默认文档地址：

- Swagger: `http://127.0.0.1:1900/docs`

### 2.2 启动 CLI（可选）

```bash
uv run python main.py
```

CLI 支持登录、聊天、会话切换、skills/profile 查看。

```json
  "password": "alice123"
```

```json

  "user_id": 3,

}
```

### 3.2 登录

`POST /auth/login`

```json
{
  "username": "alice",
  "password": "alice123"
```

## 4. 学习对话怎么用

### 4.1 发起一次学习会话

`POST /chat`

```json
{
  "session_id": "s-alice-001",
  "user_id": 3,
  "topic": "二分查找",
  "user_input": "我想学习二分查找"
}
```

返回关键字段：

- `stage`：当前阶段
- `reply`：助手回复
- `summary`：总结阶段时可用
- `citations`：检索证据（若启用 RAG）

### 4.2 多轮继续

保持同一个 `session_id`，重复调用 `/chat` 即可。  
系统会自动保存阶段并推进流程。

## 5. 如何添加 RAG 信息（重点）

RAG 入库接口：`POST /knowledge/ingest`

### 5.1 添加全局知识（global）

适合教材、公共资料，所有用户可检索：

```json
{
  "source_type": "text",
  "scope": "global",
  "topic": "二分查找",
  "title": "算法导论",
  "content": "二分查找依赖有序数组，每次比较中间元素并缩小搜索区间。"
}
```

### 5.2 添加个人知识（personal）

适合个人错题、个人总结，必须带 `user_id`：

```json
{
  "source_type": "text",
  "scope": "personal",
  "user_id": 3,
  "topic": "二分查找",
  "title": "我的错题本",
  "content": "我经常把 left/right 边界更新写错。"
}
```

### 5.3 添加图片知识（image + OCR）

```json
{
  "source_type": "image",
  "scope": "global",
  "topic": "图论",
  "image_id": "img-001",
  "title": "图论笔记",
  "content": "这是OCR提取前的图片内容占位或文本"
}
```

系统会先做 OCR 文本化，再按统一 RAG 链路切块入库。

### 5.4 直接上传文件入库（推荐）

接口：`POST /knowledge/ingest-file`（`multipart/form-data`）

支持文件类型：

- `txt`
- `docx`
- `pdf`
- `png`
- `jpg/jpeg`

示例（curl）：

```bash
curl -X POST "http://127.0.0.1:1900/knowledge/ingest-file" \
  -F "file=@D:/docs/algorithm.pdf" \
  -F "scope=global" \
  -F "topic=算法" \
  -F "title=算法讲义"
```

个人知识示例：

```bash
curl -X POST "http://127.0.0.1:1900/knowledge/ingest-file" \
  -F "file=@D:/docs/my_notes.docx" \
  -F "scope=personal" \
  -F "user_id=3" \
  -F "topic=二分查找"
```

说明：

- 未显式传 `source_type` 时，后端会根据扩展名自动推断。
- 图片文件会走 OCR，再按文本链路切块入库。
- `scope=personal` 时必须传 `user_id`。

## 6. 如何检索 RAG 信息

检索接口：`POST /knowledge/retrieve`

### 6.1 检索全局知识

```json
{
  "query": "有序数组如何查找",
  "scope": "global",
  "topic": "二分查找",
  "top_k": 3
}
```

### 6.2 检索个人知识

```json
{
  "query": "边界条件",
  "scope": "personal",
  "user_id": 3,
  "topic": "二分查找",
  "top_k": 3
}
```

返回里会包含：

- 命中文本 `text`
- 分数 `score`
- 各算法分数（`bm25_score`, `rrf_score`, `vector_score`, `rerank_score`...）
- 元数据（`scope`, `user_id`, `title`, `chapter`, `page_no` 等）

## 7. 在聊天中使用 RAG 的方式

你不需要在 `/chat` 里手工指定检索参数。  
只要后端启用 `RAG_ENABLED=true`，系统会自动按当前输入检索证据并注入到提示词。

你可以在 `/chat` 响应中查看 `citations`：

- 看命中了哪些证据
- 看证据来自 global 还是 personal
- 看由哪个 tool 提供（`tool` 字段）

## 9. 常见问题

### Q1: `scope=personal` 报 400？

因为 personal 必须传 `user_id`。

### Q2: 为什么聊天里没有 citations？

检查：

- `RAG_ENABLED` 是否为 `true`
- 当前 query 是否有可命中知识
- 你是否把知识写到对应的 scope/topic/user_id

### Q3: global 与 personal 的区别？

- global：公共知识，所有用户共享
- personal：私有知识，严格按 user_id 隔离

---

## 10. 推荐实践

- 教材/标准资料写入 `global`
- 个人错题/经验写入 `personal`
- 聊天与个人检索统一使用同一 `user_id`
- topic 尽量稳定、规范，提升检索命中率
