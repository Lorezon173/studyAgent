## 1. 项目概述

### 1.1 项目定位

LearningAgent 是一个基于 LangGraph 的智能学习代理系统，采用费曼学习法实现多轮对话教学闭环。系统支持：
- 多轮对话状态管理
- RAG 知识检索与问答
- 学习进度追踪与掌握度评估
- 会话持久化与历史记录

### 1.2 核心特性

| 特性 | 描述 |
|------|------|
| LangGraph 编排 | 使用状态图实现复杂多轮对话流程 |
| 条件边路由 | 根据意图、阶段、诊断结果动态路由 |
| 检查点持久化 | SQLite 后端支持会话状态恢复 |
| 重试策略 | 节点级容错，LLM/RAG/DB 独立重试配置 |
| RAG 混合检索 | BM25 + 向量 + Rerank 多路召回 |
| Langfuse 可观测性 | 全链路追踪，LLM/RAG/Tool 执行监控 |

## 3. 核心模块详解

### 3.1 Agent 模块 (`app/agent/`)

#### 3.1.1 状态定义 (`state.py`)

```python
class LearningState(TypedDict, total=False):
    # 会话标识
    session_id: str
    user_id: Optional[int]
    
    # 对话状态
    topic: Optional[str]          # 当前学习主题
    stage: str                    # 阶段: start -> explained -> followup_generated
    user_input: str               # 用户输入
    
    # 内容字段
    diagnosis: str                # 诊断结果
    explanation: str              # 讲解内容
    restatement_eval: str         # 复述评估
    followup_question: str        # 追问
    summary: str                  # 总结
    
    # 路由控制
    intent: str                   # 意图: teach_loop, qa_direct, review, replan
    has_history: bool             # 是否有历史记录
    rag_found: bool               # RAG 是否命中
    
    # 追踪
    history: List[str]
    branch_trace: List[dict]
    citations: List[dict]
```

#### 3.1.2 图结构 (`graph_v2.py`)

```
                    ┌─────────────────┐
                    │  intent_router  │ (入口)
                    └────────┬────────┘
                             │
          ┌──────────────────┼──────────────────┐
          ▼                  ▼                  ▼
   ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
   │history_check │   │   rag_first  │   │   replan     │
   └──────┬───────┘   └──────┬───────┘   └──────────────┘
          │                  │
    ┌─────┴─────┐     ┌─────┴─────┐
    ▼           ▼     ▼           ▼
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│diagnose│ │explain │ │rag_ans │ │llm_ans │
└───┬────┘ └───┬────┘ └────────┘ └────────┘
    │          │
    ▼          ▼
┌────────┐ ┌────────┐
│retrieve│ │restate │
└───┬────┘ └───┬────┘
    │          │
    ▼          ▼
┌────────┐ ┌────────┐
│ explain│ │followup│
└───┬────┘ └───┬────┘
    │          │
    ▼          ▼
┌────────┐ ┌────────┐
│restate │ │ summary│──▶ END
└────────┘ └────────┘
```

#### 3.1.3 节点函数 (`nodes.py`)

| 节点 | 功能 | 重试策略 |
|------|------|----------|
| `intent_router_node` | 意图识别与路由 | LLM_RETRY |
| `history_check_node` | 历史记录检查 | DB_RETRY |
| `diagnose_node` | 学习诊断 | LLM_RETRY |
| `explain_node` | 内容讲解 | LLM_RETRY |
| `restate_check_node` | 复述检测 | LLM_RETRY |
| `followup_node` | 追问生成 | LLM_RETRY |
| `summarize_node` | 总结生成 | LLM_RETRY |
| `rag_first_node` | RAG 优先检索 | RAG_RETRY |
| `rag_answer_node` | RAG 问答 | LLM_RETRY |
| `llm_answer_node` | LLM 直答 | LLM_RETRY |
| `knowledge_retrieval_node` | 知识检索 | RAG_RETRY |

#### 3.1.4 路由函数 (`routers.py`)

| 路由 | 决策逻辑 |
|------|----------|
| `route_by_intent` | 根据 intent 路由到不同分支 |
| `route_after_history_check` | 有历史则询问，否则开始诊断 |
| `route_after_choice` | 用户选择复习或继续 |
| `route_after_diagnosis` | 根据诊断结果决定路径 |
| `route_after_restate` | 复述通过则追问，否则继续讲解 |
| `route_after_rag` | RAG 命中则用 RAG 回答，否则 LLM |

### 3.2 服务模块 (`app/services/`)

#### 3.2.1 LLM 服务 (`llm.py`)

```python
class LLMService:
    def invoke(self, system_prompt: str, user_prompt: str, 
               stream_output: bool = False) -> str:
        """带重试和流式输出的 LLM 调用"""
        
    def route_intent(self, user_input: str) -> str:
        """意图识别"""
        
    def detect_topic(self, user_input: str, current_topic: str) -> str:
        """主题检测"""
        
    def answer_direct(self, user_input: str, topic: str) -> str:
        """直接问答"""
```

#### 3.2.2 RAG 服务 (`rag_store.py`)

**混合检索流程：**
```
Query → BM25 检索 ──┬──→ RRF 融合 ──→ Rerank ──→ Top-K
       → 向量检索 ─┘
```

**核心函数：**
- `ingest_knowledge()` - 知识入库（文本分块 + 向量化）
- `retrieve_knowledge_by_scope()` - 按范围检索（global/personal）

#### 3.2.3 Agent 服务 (`agent_service.py`)

```python
class AgentService:
    def run(self, session_id: str, topic: str, user_input: str,
            user_id: int = None) -> LearningState:
        """主入口：根据配置选择 V1 或 V2 图"""
        
    def run_with_graph_v2(self, ...) -> LearningState:
        """V2 图执行：条件边 + 检查点 + Langfuse 追踪"""
```

## 4. 数据模型

### 4.1 会话状态流转

```
start → diagnosed → explained → followup_generated → summarized
  │         │           │              │
  │         │           │              └──▶ 总结生成
  │         │           └──▶ 复述检测
  │         └──▶ 讲解生成
  └──▶ 诊断完成
```

### 4.2 用户画像

存储于 `data/users.db`（SQLite）：
- 用户基本信息
- 学习历史
- 掌握度趋势
- 常见错误
